# Token Efficiency Rules

- Never read a file unless directly needed for the current task
- Do not summarize what you just did — act, then wait
- Do not explain your plan before executing it; just execute
- Prefer editing existing code over rewriting entire files
- Never output file content back unless explicitly asked
- Use the minimum number of tool calls to complete a task
- If you need to explore, use grep/search — never open files blindly
- Do not repeat context already visible in the conversation
- Avoid adding comments to code unless asked
- Give short answers in ask mode — no padding or preamble
